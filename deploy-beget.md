# Деплой лендинга на Beget

## Что нужно
- Аккаунт на [beget.com](https://beget.com) (shared-хостинг от ~100₽/мес)
- Домен `pohilko.ru` (или другой), привязанный к хостингу

---

## Шаг 1: Создать сайт в панели Beget

1. Войти в [cp.beget.com](https://cp.beget.com)
2. **Сайты** → **Добавить сайт** → указать домен `pohilko.ru`
3. **Домены** → привязать домен к сайту (если домен на стороннем регистраторе — прописать NS Beget: `ns1.beget.com`, `ns2.beget.com`)
4. Включить **бесплатный SSL** (Let's Encrypt) в разделе **Домены** → иконка замка

---

## Шаг 2: Загрузить файлы

Через **Файловый менеджер** (cp.beget.com) или по FTP/SFTP.

Загрузить в папку `pohilko.ru/public_html/`:

```
public_html/
├── .htaccess
├── index.html
├── landing.html
├── admin.html
├── privacy.html
├── photo-hero.png
├── photo-hero.webp
├── robots.txt
└── sitemap.xml
```

### FTP-доступ (если нужен)
- Хост: `pohilko.ru` (или IP из панели)
- Логин/пароль: из раздела **FTP** в панели Beget
- Папка: `/pohilko.ru/public_html/`

---

## Шаг 3: Проверить

1. Открыть `https://pohilko.ru` — должен загрузиться лендинг
2. Открыть `https://pohilko.ru/admin.html` — должна работать админка
3. Открыть `https://pohilko.ru/privacy.html` — политика конфиденциальности
4. Проверить что `http://` редиректит на `https://`
5. Проверить что `.env` и `.sql` файлы НЕ доступны (должна быть ошибка 403)

---

## Шаг 4: DNS (если домен на стороннем регистраторе)

Вариант A — **NS-записи** (рекомендуется):
```
ns1.beget.com
ns2.beget.com
```

Вариант B — **A-запись**:
```
A  @  IP-адрес-из-панели-Beget
A  www  IP-адрес-из-панели-Beget
```

DNS обновляется до 24 часов (обычно 15-60 минут).

---

## Что НЕ загружать

Эти файлы НЕ нужны на хостинге:
- `.env` — содержит секретные ключи
- `supabase-setup.sql` — схема БД
- `*.md` файлы — документация
- `.git/` — репозиторий
- `.claude/` — конфиг Claude
- `31f72683-*.png` — неиспользуемый ассет
- `ChatGPT Image *.png` — неиспользуемый ассет

---

## Заметки

- **SSL бесплатный** — Beget даёт Let's Encrypt автоматически
- **Supabase продолжает работать** — ключи зашиты в HTML, бэкенд на облаке Supabase
- **Админка** — доступна по прямой ссылке, защищена паролем через Supabase Auth
- **Обновление сайта** — просто перезалить изменённые HTML-файлы через файловый менеджер
