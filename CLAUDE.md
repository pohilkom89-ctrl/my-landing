## Стек и ограничения

- **Один файл:** всё в `landing.html` — без npm, без фреймворков, без сборщиков
- **Иконки:** только встроенные SVG (`<svg>` прямо в HTML)
- **Стили:** `<style>` внутри `landing.html`
- **JS:** `<script>` внутри `landing.html`
- **Внешние зависимости:** запрещены (ни CDN, ни шрифты по ссылке, ни аналитика)
- **Фото:** встроено как base64 в тег `<img src="...png">` (файл лежит рядом)

## Дизайн-система

```css
:root {
  --bg:      #0d0d14;   /* основной фон */
  --bg2:     #13131f;   /* фон чётных секций */
  --bg3:     #1a1a2e;   /* фон секции "Обо мне" */
  --card:    #16162a;   /* фон карточек */
  --accent:  #6c63ff;   /* акцентный цвет (кнопки, иконки) */
  --accent2: #a78bfa;   /* светлый акцент (заголовки, цены) */
  --text:    #f0f0f8;   /* основной текст */
  --muted:   #8888aa;   /* приглушённый текст */
  --border:  #2a2a40;   /* границы карточек */
  --radius:  16px;      /* скругление */
}
```

**Правила дизайна:**
- Тёмная тема — не менять на светлую
- Шрифт: `'Segoe UI', system-ui, -apple-system, sans-serif`
- Mobile-first адаптация обязательна (медиазапросы в блоке `/* MOBILE */`)
- Анимации: `transition` на hover-эффектах, `IntersectionObserver` для scroll-reveal
- Градиенты: `linear-gradient(135deg, #fff 40%, var(--accent2))` для заголовков

## Git-правила

- Коммиты **только на английском**, формат: `feat:` / `fix:` / `docs:` / `style:`
- Каждое логическое изменение — **отдельный коммит**
- **Не делать** `git push --force`
- **Не коммитить** без явной просьбы пользователя

## Правила работы с кодом

1. **Читать перед правкой** — всегда открывать `landing.html` перед изменениями
2. **Не добавлять секции** без явного запроса пользователя
3. **Не менять** цены, тексты, контакты без явного указания
4. **Минимум кода** — решать задачу минимально необходимыми изменениями
5. **После правок** — мысленно проверить мобильную вёрстку (медиазапросы)
6. **SVG только встроенные** — никаких `<img src="icon.svg">` или внешних ссылок
